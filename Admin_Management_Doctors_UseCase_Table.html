Admin Management Doctors Use Case

Use Case 7: Admin Management Doctors (UC-007)

Use Case 7 - (UC-007)
Admin Management Doctors
Related RequirementsFR-004, FR-014, FR-021
Initiating Actor:Administrator
Actor's Goal:Manage doctor accounts, approve registrations, monitor performance, and maintain doctor database integrity
Participating Actors:Doctor, Email System, Notification System
Preconditions:Administrator is logged in with admin privileges and doctor management permissions
Postconditions:Doctor accounts are properly managed, verified, and system integrity is maintained
Flow of Events for Main Success Scenario:
——→ 1. Administrator accesses doctor management dashboard.
←—— 2. System displays list of all doctors with status, specialties, and statistics.
——→ 3. Administrator selects doctor to manage or reviews pending registrations.
←—— 4. System shows detailed doctor profile with credentials and verification status.
——→ 5. Administrator performs action (approve, suspend, activate, update specialties).
←—— 6. System validates action and updates doctor status in database.
←—— 7. System sends notification to doctor about status change.
←—— 8. System logs administrative action for audit trail.
Flow of Events for Extensions (Alternate Scenarios):
3. Administrator reviews new doctor registration.
←—— System displays registration details and required documents.
——→ 4. Administrator verifies medical credentials and certifications.
←—— 5. System provides verification tools and external database access.
——→ 6. Administrator approves or rejects registration with reasons.
←—— 7. System updates status and notifies doctor of decision.

5. Administrator attempts to suspend doctor with active appointments.
←—— System shows warning about active appointments and suggests alternatives.
——→ 6. Administrator chooses to reschedule appointments or proceed with suspension.
←—— 7. System handles appointments appropriately and completes action.

4. Doctor profile contains incomplete or invalid information.
←—— System highlights missing fields and validation errors.
——→ 5. Administrator contacts doctor for required information updates.
←—— 6. System tracks communication and pending requirements.

Key Administrative Functions:

  • Registration Management: Approve/reject new doctor applications
  • Account Status Control: Activate, suspend, or deactivate doctor accounts
  • Credential Verification: Validate medical licenses and certifications
  • Profile Maintenance: Update doctor information and specialties
  • Performance Monitoring: Track doctor activities and patient feedback
  • Appointment Coordination: Handle scheduling conflicts during status changes
  • Audit Trail Management: Maintain comprehensive logs of all administrative actions
